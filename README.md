# CostSnap 🧾

**ถ่ายรูปใบกำกับภาษี → ให้ AI อ่านค่า → บันทึกต้นทุนสินค้า**

เว็บแอป (PWA) สำหรับร้านค้า/SME ไทย ใช้ Claude Vision อ่านใบกำกับภาษี/ใบเสร็จจากรูปถ่าย
สกัดข้อมูลเป็นโครงสร้าง ตรวจความถูกต้อง (เลขผู้เสียภาษี 13 หลัก, สมการยอดเงิน,
ความครบถ้วนตามมาตรา 86/4) แล้วบันทึกเป็นต้นทุนสินค้า พร้อมรายงานสรุป

> อ่านแผนการออกแบบและผลการวิจัยแนวทางฉบับเต็มได้ที่ [PLAN.md](./PLAN.md)

## ความสามารถ

- 📷 **สแกนจากกล้องมือถือ** — บีบอัดรูปฝั่งเครื่องก่อนส่ง AI
- 🤖 **AI อ่านค่าอัตโนมัติ** — ผู้ขาย/เลขผู้เสียภาษี/เลขที่/วันที่/รายการสินค้า/VAT/ยอดรวม
  ด้วย strict JSON schema (แปลงเลขไทย, ปี พ.ศ.→ค.ศ. ให้อัตโนมัติ)
- ✏️ **ตรวจทานก่อนบันทึกเสมอ** — ฟอร์มแก้ไขได้ทุกช่อง พร้อมป้ายเตือน validation
- ✅ **ตรวจความถูกต้อง** — check digit เลขผู้เสียภาษี, ผลรวมรายการ, สมการ VAT 7%,
  ความครบถ้วนใบกำกับภาษีเต็มรูป (บอกว่าใช้ขอคืน VAT ซื้อได้หรือไม่), เตือนใบซ้ำ
- 📊 **แดชบอร์ด** — ต้นทุนรายเดือน, VAT ซื้อขอคืนได้, Top ผู้ขาย, กราฟ 6 เดือน
- 📦 **ต้นทุนต่อสินค้า** — ต้นทุนเฉลี่ยถ่วงน้ำหนัก, ราคาล่าสุด, ช่วงราคาต่ำสุด–สูงสุด
- 💾 **ข้อมูลอยู่บนเครื่องผู้ใช้** (IndexedDB) พร้อม Export CSV (เปิด Excel ได้) และ
  Backup/Restore JSON รวมรูปภาพ

## เริ่มใช้งาน

```bash
npm install
export ANTHROPIC_API_KEY=sk-ant-...   # คีย์จาก https://platform.claude.com
npm run dev                            # เปิด http://localhost:3000
```

Deploy บน Vercel: ตั้ง Environment Variable `ANTHROPIC_API_KEY` แล้ว deploy ได้เลย

ตัวเลือกเพิ่มเติม: `EXTRACT_MODEL` เปลี่ยนโมเดลที่ใช้ (ค่าเริ่มต้น `claude-opus-4-8`)

## สถาปัตยกรรม

- **Next.js 15 (App Router) + TypeScript** — PWA มือถือเป็นหลัก
- **`/api/extract`** — เรียก Claude ด้วย strict tool use → ได้ JSON ตาม schema เสมอ
  จากนั้นตรวจ check digit + สมการเงินฝั่ง server
- **Dexie (IndexedDB)** — ตาราง `receipts` (พร้อมรูปต้นฉบับ) และ `items`
  (รายการสินค้า normalize ชื่อเพื่อรวมต้นทุนต่อสินค้า)
- API key อยู่ฝั่ง server เท่านั้น รูปถูกส่งไปประมวลผลชั่วคราว ไม่มีฐานข้อมูลกลาง
